**Verdict:** `human_needed` — **functionally complete; one human-verification item outstanding.** All 8 phases executed (74/74 plans), all 48 requirements delivered and exercised on the live cluster, all 8 integration flows wire together. The single remaining item is **G3**: 2 interactive Phase 07 UAT tests (timer render, signal handling) that need a real terminal on the candidate's cluster — now formally tracked in `07-VERIFICATION.md` as `human_needed`.

## Gaps & Findings

### G1 — BLOCKER — Phase 07 has no VERIFICATION.md (verification-artifact)
Phase 07 shipped `07-UAT.md` (status: passed, 9/9 automated) and `07-VALIDATION.md`, but never a `07-VERIFICATION.md` or `07-SUMMARY.md`. The milestone audit has no formal verification record for the milestone's *Core Value* phase (exam mode + reporting). Functionality is evidenced by UAT; the artifact is the gap.
**Fix:** run `/gsd-verify-phase 07` (or author `07-VERIFICATION.md` from the passing UAT) so RUN-03..06, MOCK-01, MOCK-03, REPORT-01..02 have `requirements_coverage` frontmatter.

### G2 — WARNING — Phase 08 verification not structured (verification-artifact)
`08-VERIFICATION.md` exists with `status: PASS` and a body listing all 10 requirements delivered, but has no `requirements_coverage` frontmatter and no `08-SUMMARY.md`. Tooling that reads frontmatter sees Phase 08 as uncovered.
**Fix:** add `requirements_coverage` frontmatter to `08-VERIFICATION.md`; optionally backfill `08-SUMMARY.md`.

### G3 — WARNING — Two interactive UAT tests never human-verified (human-verification)
`07-UAT.md` tests 1 (timer renders/decrements/survives Ctrl-Z) and 2 (Ctrl-C flags, Ctrl-Z pauses, EXIT-trap persists) are SKIPPED — they need a real interactive terminal. RUN-04 and RUN-06 rest on automated data-level tests only; the pixel/job-control behavior is unconfirmed.
**Fix:** candidate runs `cka-sim exam blueprint-alpha`, exercises Ctrl-C / Ctrl-Z / `fg` / `--resume`, records in `07-HUMAN-UAT.md`.

### G4 — WARNING — `~/.cka-sim/history` is a dead directory (integration-contract)
`bootstrap.sh:87`, `preflight.sh:105,113`, and `doctor.sh:82` create and check `~/.cka-sim/history`, but no code writes or reads it. `exam-state.sh` writes sessions to `~/.cka-sim/sessions/`; `list.sh` and `cka-sim list history` read `sessions/`. The dir is advertised in bootstrap output but stays permanently empty — a Phase 01 ↔ Phase 06 naming mismatch.
**Fix:** either drop `history/` from the state-dir set, or point `list`/`state` at it. Low effort, no functional break.

### G5 — WARNING — `~/.cka-sim/reports` vs `~/.cka-sim/sessions` split (integration-contract)
`drill.sh:319` writes drill reports to `~/.cka-sim/reports/`; exam reports land in `~/.cka-sim/sessions/<ts>.md`. `score.sh` only reads `state::report_path` (→ `sessions/`), so `cka-sim score` silently ignores all drill output. Likely intentional (drill vs exam are separate flows) but undocumented; bootstrap messaging implies `reports/` is *the* report store.
**Fix:** document the split, or unify the report location.

### G6 — WARNING — `storage/04-csi-volumesnapshot` orphaned from blueprints (content-orphan)
`packs/storage/manifest.yaml:16` declares `storage-csi-volumesnapshot` and the question dir is complete, but neither `exams/blueprint-alpha` nor `exams/blueprint-bravo` references it. It's reachable via `cka-sim drill storage` but never appears in a mock exam, and `lint-packs.sh` pass H only checks the reverse direction.
**Fix:** add it to a blueprint draw, or accept it as drill-only and note that intent.

### G7 — WARNING — Nyquist validation debt (validation-debt)
Phases 01/02/03/08 have no `VALIDATION.md`. Phases 04/05/06/07 have one, all `status: draft` with `nyquist_compliant: false`. `07-VALIDATION.md` lists 15 per-task verifications all `⬜ pending` and `wave_0_complete: false` — though the Wave 0 test infra it calls for (`cka-sim/tests/exam/`, `tests/fixtures/exam/`, mock-pack-alpha) **does now exist on disk**, so the doc is stale rather than the work being undone.
**Fix:** reconcile the VALIDATION.md docs with built state, or run `/gsd-audit-uat`-style Nyquist closure if formal compliance is wanted for v1.0 sign-off.
