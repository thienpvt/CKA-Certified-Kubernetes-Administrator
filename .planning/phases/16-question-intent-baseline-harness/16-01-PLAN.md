         command -v jq >/dev/null 2>&1 || { err "jq not found in PATH"; exit 2; }
         command -v python3 >/dev/null 2>&1 || { err "python3 not found in PATH"; exit 2; }

          local tsv_tmp=$(mktemp)

          # for the lint-style per-row err lines. We swallow stderr because audit
          # has its own table renderer and doesn't want the verbose lint output.
          { cka_sim::symptom_diff::run_one "$yaml_file" "$q_dir" "$pack" "$q_name" "audit" 3>"$tsv_tmp"; } 2>/dev/null || rc=$?
          ```

       b. Read the TSV rows back. If `rc == 0` (clean): print one line `<glyph_pass> <pack>/<q_name>: PASS (N/N expectations met)` where N is the count of rows with verdict=PASS. Glyph: `✓` (UTF-8).

          - Header line: `✗ <pack>/<q_name>: FAIL (M expectation(s) failed of N)`.
          - Flat table with header row `kind | name | jsonpath | claimed | actual | verdict` and one row per TSV entry. Pad column widths so columns align; verdict glyph: `✓` PASS, `✗` FAIL, `?` MISSING (resource absent), `!` ERROR (kubectl/jq error). Use a small awk one-liner to compute col widths and render.
          - **Claim source block:** for each FAIL row, extract a 3-line excerpt around the relevant prose line in `<q_dir>/question.md`. The line number comes from the TSV `line` field (which is already the line in `expected-symptom.yaml`); for the `Claim source:` block we need the line in `question.md` instead. Implementation: use `grep -n` to find the YAML's `name:` value as a substring inside `question.md`, take the first match, output `±2 lines` around it via `sed -n 'A,Bp'`. Format:

                The PVC named app-data is stuck in Pending state. The Pod

            ```
