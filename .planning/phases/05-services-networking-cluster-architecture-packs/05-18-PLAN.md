<objective>
Close UAT gap 2 (test 7 — S&N Q06 netpol-endport ref-solution grades 5/6 instead of 6/6).

Problem: `seed_netpol_skeleton` (from `lib/setup.sh`) emits a baseline NetworkPolicy selecting `app=q06-client` with `policyTypes: [Ingress, Egress]` and only ONE egress rule — UDP/53 + TCP/53 to kube-system. All other egress from q06-client is denied. The ref-solution creates `q06-allow-range` as an Ingress-only policy on the server side; it does NOT relax client egress. Grader assertion 5 `kubectl exec q06-client -- wget q06-server:8085` then times out because q06-client's egress to q06-server is blocked by the baseline. Assertion 6 (8095 must fail) passes coincidentally via the baseline block.

Fix: after `cka_sim::setup::seed_netpol_skeleton` runs in setup.sh, apply a supplemental NetworkPolicy `q06-client-egress` that ADDS a TCP 8080-8090 egress allowance from `app=q06-client` to `app=q06-server` (same namespace). The 8095 probe still fails because endPort tops out at 8090 — assertion 6 survives.

Broken-state behaviour (updated for truth #4): NetworkPolicy is additive and scoped by podSelector. `seed_netpol_skeleton` selects `app=q06-client` only; the new `q06-client-egress` also selects `app=q06-client`. NEITHER attaches an ingress policy to q06-server — so q06-server's ingress is default-allow in broken state. Once `q06-client-egress` authorises TCP 8080-8090 egress from q06-client, assertion 5 (wget q06-server:8085) SUCCEEDS in broken state (both directions allowed by absence of matching deny). Assertion 6 (wget q06-server:8095) FAILS in broken state because client-side egress tops out at 8090. Assertions 1-4 all fail (no q06-allow-range resource to inspect → missing protocol declaration fires `netpol-endport-missing-protocol` trap). Net broken score: 2/6 rc=1 with the trap recorded. Post-ref-solution score: 6/6 rc=0.

The baseline remains untouched (no parameter added to `seed_netpol_skeleton` — that helper is shared with future questions per 05-01-SUMMARY.md and widening its signature here would ripple). A sibling NetworkPolicy is the correct shape: two overlapping egress policies are UNIONed by k8s ingress/egress admission (both allow-listed egresses are granted).

Current setup.sh (post-P07) seeds three resources in the lab ns:
- Pod `q06-server` (listens TCP 8080-8090) with label `app=q06-server`
- Service `q06-server` (ports 8080, 8085, 8090, 8095)
- Pod `q06-client` with label `app=q06-client`

Then calls `cka_sim::setup::seed_netpol_skeleton "$CKA_SIM_LAB_NS" q06-baseline app=q06-client`, which creates a NetworkPolicy:
```
metadata.name: q06-baseline
spec.podSelector.matchLabels.app: q06-client
spec.policyTypes: [Ingress, Egress]
spec.egress:
  - to: [{ namespaceSelector: { matchLabels: { kubernetes.io/metadata.name: kube-system } } }]
    ports: [{ protocol: UDP, port: 53 }, { protocol: TCP, port: 53 }]
```

Grader (post-P07, unchanged by this plan) asserts 6 items:
1-3. `q06-allow-range` NetworkPolicy exists + port=8080 + endPort=8090 (ingress side, candidate's task)
4. NetworkPolicy declares protocol TCP
5. `kubectl exec q06-client -- wget q06-server:8085` succeeds (in-range probe)
6. `kubectl exec q06-client -- wget q06-server:8095` FAILS (out-of-range probe)

Ref-solution creates:
```
kind: NetworkPolicy
metadata.name: q06-allow-range
spec.podSelector.matchLabels.app: q06-server
spec.policyTypes: [Ingress]
spec.ingress:
  - from: [{ podSelector: { matchLabels: { app: q06-client } } }]
    ports: [{ protocol: TCP, port: 8080, endPort: 8090 }]
```

NetworkPolicy union semantics (K8s docs): a pod is affected by the UNION of all policies that select it. For traffic to flow:
- Server-side ingress rule must match. In broken state no policy selects q06-server → server ingress is default-allow. After ref-solution `q06-allow-range` attaches ingress filter to q06-server.
- Client-side egress rule must match. Baseline `q06-baseline` selects q06-client and permits only DNS egress; the new `q06-client-egress` adds TCP 8080-8090 egress to app=q06-server. Union → egress allowed for 8080-8090 and DNS; 8095 denied.

Both conditions combined: broken state has client-side egress allowance for 8080-8090 AND no server-side ingress filter → 8085 traffic flows (assertion 5 passes in broken). After ref-solution, both sides are explicitly allow-listed for 8080-8090 (6/6).

Gap 2 fix shape (from UAT.md missing):
> In setup.sh, after seed_netpol_skeleton, apply an additional NetworkPolicy (or extend baseline) selecting app=q06-client with an egress rule: to podSelector app=q06-server, TCP ports 8080-8090 via endPort.

Using a second NetworkPolicy rather than extending `seed_netpol_skeleton` is the stated preference — the helper is shared with future questions per 05-01-SUMMARY.md's helper signature lock.
