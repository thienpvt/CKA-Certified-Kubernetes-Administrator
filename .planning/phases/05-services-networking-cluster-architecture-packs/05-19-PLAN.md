<objective>
Close UAT gap 3 (test 9 — CA Q02 etcd-backup-restore ref-solution grades 1/3 instead of 3/3).

Problem: setup.sh at lines 15-16 pre-creates `$sandbox/restored-data/` plus a `.pre-existing` sentinel file. `etcdutl snapshot restore` refuses to write into an existing data-dir and aborts under `set -euo pipefail`, so ref-solution.sh exits before it can populate `restored-data/member/wal/`. Grader assertion 3 (`restored member/wal directory exists`) then fails. Secondary issue: setup.sh line 23 only `warn`s when `etcdutl` is missing; on a v1.35 kubeadm node with only `etcdctl` installed, both the grader's `etcdutl snapshot status` assertion AND ref-solution's `etcdutl snapshot restore` silently fail — exactly the observed 1/3 outcome.

Fix per UAT.md gap 3 missing items:
1. Drop the `mkdir -p "$sandbox/restored-data"` and `touch "$sandbox/restored-data/.pre-existing"` lines from setup.sh so the dir is absent when etcdutl restore runs. Keep `$sandbox/.cka-sim-sentinel` and the sandbox itself.
2. Promote the etcdutl presence check from `warn` to `die` so the drill fails fast with a clear diagnostic on a node missing etcdutl.
3. Harden ref-solution.sh with `rm -rf "$sandbox/restored-data"` immediately before `etcdutl snapshot restore` — protects against stale dirs from earlier aborted runs that reset.sh may not have cleaned if sentinel was absent.

grade.sh is NOT touched. Its trap-detection greps on `$sandbox/apply-script.sh` (setup writes the TODO stub; candidate or ref-solution overwrites with the real commands) remain correct. The grader also still calls `etcdutl snapshot status` — but now setup's die guarantees etcdutl is present, so that assertion only fails when the candidate's script is actually broken.

question.md, metadata.yaml, reset.sh are NOT touched.

Purpose: make Q02 reach 3/3 rc=0 after ref-solution on any 1+2 kubeadm v1.35 node that has etcd-client v3.5+ installed with etcdutl.

    Context — the file has two places that matter:
    1. The `cat > "$sandbox/apply-script.sh" <<'EOF' ... EOF` block that writes the candidate-facing script. This block's body is heredoc-literal (quoted EOF); the `rm -rf` should appear INSIDE the heredoc too, BEFORE the `etcdutl snapshot restore` line inside the script. That makes the written apply-script self-contained — a candidate running the script later gets the same protection.
    2. The direct `etcdutl snapshot restore ...` call at the bottom of ref-solution.sh (outside the heredoc, so the ref-solution itself runs it live). Add `rm -rf "$sandbox/restored-data"` immediately before this line too.

    Both insertions are required because:
    - The heredoc version ships in the generated apply-script.sh that grade.sh's trap detector also consumes; including `rm -rf` there keeps the script genuinely runnable on re-invocation.
    - The direct-invocation version is what ref-solution actually executes during `--ref-solution`; without it the live run still fails on stale restored-data/ (gap 3 scenario after a partial failure).

    Keep `set -euo pipefail` and every other line intact. Do NOT remove the existing `ETCDCTL_API=3 etcdctl ... snapshot save` call. Do NOT change `--data-dir="$sandbox/restored-data"` — that exact path is what grade.sh's trap detector expects. Do NOT touch `/var/lib/etcd` or any live etcd state.

    Sanity — the final ref-solution.sh shape (fragment):
    ```
    cat > "$sandbox/apply-script.sh" <<'EOF'
    #!/bin/bash
    set -euo pipefail
    export ETCDCTL_API=3
    ETCDCTL_API=3 etcdctl \
      --endpoints=https://127.0.0.1:2379 \
      --cacert=/etc/kubernetes/pki/etcd/ca.crt \
      --cert=/etc/kubernetes/pki/etcd/server.crt \
      --key=/etc/kubernetes/pki/etcd/server.key \
      snapshot save /tmp/q02-etcd-backup/snapshot.db
    rm -rf /tmp/q02-etcd-backup/restored-data          ← NEW inside heredoc
    etcdutl snapshot restore /tmp/q02-etcd-backup/snapshot.db \
      --data-dir=/tmp/q02-etcd-backup/restored-data

    ETCDCTL_API=3 etcdctl \
      --endpoints=https://127.0.0.1:2379 \
      --cacert=/etc/kubernetes/pki/etcd/ca.crt \

      --key=/etc/kubernetes/pki/etcd/server.key \
    ```
