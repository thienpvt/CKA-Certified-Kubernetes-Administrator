## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer commit → GHA shellcheck job | A developer pushes shell or YAML that is then linted on `ubuntu-latest`. Trust = high (we are the developer); the lint gate exists to catch our own mistakes, not adversarial input. |
| `.shellcheckrc` rule disables → CI lint coverage | Each repo-global disable widens what slips past lint. Each disable MUST carry a rationale or it cannot be reviewed. |
| out-of-scope find exclusions → CI coverage | Each `-not -path '<glob>'` exclusion narrows the corpus that gets linted. Each exclusion MUST name the rationale (vendored, generated, 3rd-party). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-01-01 | Tampering | shell-injection in dispositioned files | mitigate | fix-in-code dispositions for SC2086 / SC2046 (unquoted variables, unquoted command substitution) close real word-split / glob-expansion holes. Per-finding edits enforce single-quoting / arrays where appropriate. |
| T-23-01-02 | Repudiation | undocumented lint relaxation | mitigate | Every relax-with-rationale entry carries a one-line rationale comment (in `.shellcheckrc` or per-line). 23-01-FINDINGS.md preserves the audit trail; 23-01-SUMMARY.md transcribes the table for milestone audit. |

| T-23-01-05 | Elevation of Privilege | `.shellcheckrc` global disable masking a real bug class | mitigate | Disposition rule ordering (mandatory): prefer fix-in-code over relax. relax-with-rationale is reserved for idiom fights (SC2155 on `local x=$(cmd)`) and dynamic-source patterns (SC1091); never used for SC2086/SC2046-class word-split bugs.
