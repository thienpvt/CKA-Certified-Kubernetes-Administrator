Ship the shared setup helper library `cka-sim/lib/setup.sh` that every new Phase 4 question (and the two retrofitted Phase 3 reference questions in Plan 03) sources. Extract the 120s ns-Active wait from Phase 3 commit `5c421c1` into a single helper so the fix propagates automatically and is never duplicated again.

Exports exactly 4 functions (RESEARCH §4 contract):
- `cka_sim::setup::ensure_lab_ns <ns> <pack> <question-id>` — idempotent Namespace apply with cka-sim labels.
- `cka_sim::setup::wait_for_ns_active <ns> <pack> <question-id> [<timeout-seconds>]` — poll up to timeout (default 120s) for `status.phase == Active`; re-apply the Namespace if it disappears mid-wait (reset --wait=false race absorber).

- `cka_sim::setup::seed_deployment <ns> <name> <image> [--replicas N] [--sa SA] [--cpu X] [--memory Y]` — minimal Deployment with opt-in SA and requests.

Purpose: kill the Phase 3 duplicated-wait regression surface, unblock parallel authoring in Wave 3.

    **File 10 — `cka-sim/tests/cases/setup_helpers_wait_for_ns_active.sh`**:
    ```
    #!/bin/bash
    # Verifies wait_for_ns_active returns 0 on Active; re-applies ns on empty phase.
    set -uo pipefail
    : "${CKA_SIM_ROOT:?must be set by run.sh}"
    # shellcheck source=../lib/assert.sh disable=SC1091
    source "$CKA_SIM_ROOT/tests/lib/assert.sh"

    case_failed=0

    # Counters for kubectl get and kubectl apply invocations.
    get_count=0; apply_count=0
    phase_sequence=("" "Active")   # first call returns empty (triggers re-apply), second returns Active
    kubectl() {
      case "${1:-}" in
        get) get_count=$(( get_count + 1 )); printf '%s' "${phase_sequence[$(( get_count - 1 ))]:-Active}" ;;
        apply) apply_count=$(( apply_count + 1 )); cat >/dev/null ;;  # swallow heredoc
        *) return 64 ;;
      esac
    }
    export -f kubectl

    # Shorten sleep to 0s for test speed.
    sleep() { :; }
    export -f sleep

    # shellcheck source=../../lib/setup.sh disable=SC1091
    source "$CKA_SIM_ROOT/lib/setup.sh"

    # First call triggers re-apply (empty phase), second call returns Active.
    if ! cka_sim::setup::wait_for_ns_active "cka-sim-storage-01" "storage" "storage-pvc-binding" 10; then
      case_failed=1
    fi
    expect_eq "$apply_count" "1" "re-apply fired exactly once on empty phase" || case_failed=1
    exit "$case_failed"
    ```
