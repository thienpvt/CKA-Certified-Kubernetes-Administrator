    Create `cka-sim/scripts/lint-question-symptom.sh` (mode 0755). Structure:

    1. **Header** (mirror lint-traps.sh):

       ```bash
       #!/bin/bash
       # cka-sim/scripts/lint-question-symptom.sh — Phase 15 CI-01.
       ```

       ```bash
       declare -A KIND_ALIAS=(
         [pvc]=persistentvolumeclaim
         [pv]=persistentvolume
         [pod]=pod
         [svc]=service
         [deploy]=deployment
         [networkpolicy]=networkpolicy
         [configmap]=configmap
       ```

    5. **Per-question loop** — driver:
       ```bash
       errors=0
       checked=0
       target_arg="${1:-}"   # optional pack/question filter, e.g. storage/01-pvc-binding

       while IFS= read -r yaml_file; do
         q_dir="$(dirname "$yaml_file")"
         pack="$(basename "$(dirname "$q_dir")")"
         q_name="$(basename "$q_dir")"
         [[ -n "$target_arg" && "$pack/$q_name" != "$target_arg" ]] && continue
         checked=$(( checked + 1 ))

         info "==> $pack/$q_name"
         _diff_one_question "$yaml_file" "$q_dir" "$pack" "$q_name" || errors=$(( errors + 1 ))
       done < <(find "$CKA_SIM_ROOT/packs" -name 'expected-symptom.yaml' -type f | sort)
       ```

    6. **`_diff_one_question`** function (the core):
       a. Compute a unique CKA_SIM_LAB_NS: `local ns="cka-sim-lint-${pack}-${q_name}"` (truncate to <=63 chars, replace non-RFC1123 chars with '-').
       b. `export CKA_SIM_LAB_NS="$ns"` and run setup.sh in a subshell to avoid polluting parent env: `( bash "$q_dir/setup.sh" ) || { err "$pack/$q_name: setup.sh failed"; return 1; }`.
       c. Wait briefly (kubectl wait or 30s sleep cap) for resources to settle. Use `kubectl wait` only when expected resources include conditions; otherwise rely on the per-question setup's own waits.
       d. Parse the YAML via a python3 helper that emits a flat shell-friendly form:
          ```bash
          parsed="$(python3 - "$yaml_file" "$ns" <<'PY'
          import sys, yaml
          path, ns = sys.argv[1], sys.argv[2]
          with open(path) as f:
            d = yaml.safe_load(f)
          # Substitute ${CKA_SIM_LAB_NS}
          def sub(v):
            if isinstance(v, str):
              return v.replace('${CKA_SIM_LAB_NS}', ns)
            return v
          for r in (d.get('resources') or []):
            print('R', r.get('kind'), r.get('name'), sub(r.get('namespace') or d.get('namespace') or ''), sep='\t')
            for jp, ev in (r.get('expect') or {}).items():
              print('E', r.get('kind'), r.get('name'), jp, sub(str(ev)), sep='\t')
          for r in (d.get('absent_resources') or []):
            print('A', r.get('kind'), r.get('name'), sub(r.get('namespace') or d.get('namespace') or ''), sep='\t')
          PY
          )"
          ```
       e. For each `R` line: validate kind in KIND_ALIAS; `kubectl get <canonical> <name> [-n <ns>] -o json` and stash the JSON in a tempfile keyed by `<kind>/<name>`.
       f. For each `E` line: jq the matching JSON tempfile with the jsonpath. The "jsonpath" we accept is the dot-path notation from the schema (e.g. `status.phase`, `spec.storageClassName`, `status.conditions[?(@.type=="Available")].status`). For simplicity, translate `a.b.c` to `.a.b.c` jq query; for the conditions selector, accept jq-native syntax only — `[?(@.type=="X")].status` becomes `.status.conditions[] | select(.type=="X") | .status` in the executor (the YAML records the user-friendly form; the script translates).

       Concrete translator (bash function):
       ```bash
       _jsonpath_to_jq() {
         local jp="$1"
         # Special-case the conditions[?(@.type=="X")].field form
         if [[ "$jp" =~ ^(status|spec)\.conditions\[\?\(@\.type==\"([^\"]+)\"\)\]\.([a-zA-Z_]+)$ ]]; then
           printf '.%s.conditions[] | select(.type=="%s") | .%s' "${BASH_REMATCH[1]}" "${BASH_REMATCH[2]}" "${BASH_REMATCH[3]}"
           return 0
         fi
         # Plain dotted path: status.phase -> .status.phase
         printf '.%s' "$jp"
       }
       ```

       g. Compare actual vs expected. On mismatch, find the line number of the offending field in expected-symptom.yaml via grep -n, emit:
          `err "$yaml_file:$line: $kind/$name $jp expected '$expected', got '$actual'"`
          Increment local fail counter.
       h. For each `A` line: `kubectl get <canonical> <name> [-n <ns>]` exit 0 means resource exists → fail; non-0 means absent → pass.
       i. Run `bash "$q_dir/reset.sh"` in a subshell after diffing (success or fail). Tolerate reset failure with a warn — do not let reset failure mask diff results.
       j. Return 1 if any fail accumulated; else 0.

    7. **Footer:**
       ```bash
       if (( errors > 0 )); then
         err "$errors question(s) had symptom-diff failures across $checked checked"
         exit 1
       fi
       ok "symptom diff: $checked question(s) passed"
       ```

    8. **chmod 0755** on the file.

    LF line endings. shellcheck-clean (run shellcheck locally as part of the verify step).
