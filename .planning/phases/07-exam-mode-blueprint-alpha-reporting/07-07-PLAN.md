Root cause (from debug session — five compounding defects):
1. PRIMARY — `read` not restarted after a trap. exam.sh:209 (and confirm_submit:242) treats a
   trap-interrupted non-zero `read` as EOF and ends the exam.
2. on_tstp re-arms the TSTP trap only AFTER resume and does interruptible `state::save` work →
   a nested signal re-enters trap handling and `kill -STOP $$` may never be reached.
3. on_tstp tail + on_cont both touch timer spawn/stop and pause accounting → races, double-spawned
   (orphaned) redraw_loop.
4. No `stty` save/restore around pause/resume; background timer `tput` collides with foreground
   `read` → terminal left non-echo/non-canonical, next `read` blocks invisibly.
5. setup.sh runs as a foreground child (exam.sh:170) with no `|| true` under `set -e` → an
   interrupted setup returns 130 and kills the whole exam.
</context>

<task type="auto">
  <name>Task 1: Make read restartable and contain signals during question setup</name>
  <files>cka-sim/lib/cmd/exam.sh</files>
  <action>
  Fix defects 1 and 5 — the two that directly end the exam.

  DEFECT 1 — restartable read (exam.sh:207-212 in question_loop, and :240-242 in
  confirm_submit):
  - In bash, when a trap fires during a blocking `read`, after the handler returns `read`
    exits non-zero (interrupted, status > 128) WITHOUT consuming a line. The current code
    treats ANY non-zero as EOF (`CKA_SIM_EXAM_ENDED=1; break`).
  - Replace the single `if ! read -r action` with a loop that distinguishes trap-interrupt
    from genuine EOF: call `read -r action`; capture its exit status; if status is 0 →
    proceed; if status > 128 (interrupted by signal) → loop and re-issue `read` (re-print
    the `> ` prompt); only treat status == 1 with empty input AND closed stdin as real EOF
    (`CKA_SIM_EXAM_ENDED=1; break`). A clean way: `while ! read -r action; do (( $? > 128 ))
    && continue; CKA_SIM_EXAM_ENDED=1; break 2; done` — verify the `$?` is captured before
    it is clobbered (assign `rc=$?` immediately after read). Re-print `> ` on each retry.
  - Apply the same restart logic to the `read -r confirm` in confirm_submit (:242): a
    trap-interrupt there must re-prompt, not fall through to `confirm="y"`. Keep the genuine
    EOF behavior (`|| confirm="y"`) only for true EOF, not signal-interrupt.

  DEFECT 5 — contain signals during setup (setup_question, exam.sh:156-173):
  - `bash "$qdir/setup.sh" </dev/null` at :170 has no `|| true`; under `set -e` an
    interrupted setup (rc 130) kills the exam. Wrap it so a signal during setup aborts/flags
    the CURRENT question instead of the exam: run it as `bash "$qdir/setup.sh" </dev/null ||
    setup_rc=$?` capturing the status; if `setup_rc` indicates signal-interrupt (> 128) or
    non-zero, do NOT add idx to CKA_SIM_EXAM_SETUP_IDXS, print a warn that Q$((idx+1)) setup
    was interrupted, mark the question `flagged` via cka_sim::state::set_question_status, and
    return non-zero from setup_question. Caller (question_loop, :198) must check the return:
    on non-zero, skip present_question for this idx and advance / re-loop rather than die.
  - The `reset.sh` call at :169 already has `|| true` — leave it.
  - Do not weaken `set -euo pipefail` globally; contain only this invocation.
  </action>

  </verify>
  <done>
  `bash -n` parses clean. The question_loop read is inside a retry loop that re-prompts on

<task type="auto">
  <name>Task 2: Re-entrancy-safe TSTP/CONT handlers + single pause/resume path + stty hygiene</name>
  <files>cka-sim/lib/cmd/exam.sh, cka-sim/lib/exam-timer.sh</files>
  <action>
  Fix defects 2, 3, 4 — the nested-signal hang and terminal corruption.

  DEFECT 2+3 — self-contained, re-entrancy-safe on_tstp; consolidate pause/resume
  (exam.sh on_tstp :72-79, on_cont :81-86):
  - Add a re-entry guard global: `declare -g CKA_SIM_EXAM_IN_SIGHANDLER=0`. At the top of
    on_tstp, if it is already 1, `return` immediately. Otherwise set it to 1.
  - Inside on_tstp, BLOCK further INT and TSTP for the handler's duration:
    `trap '' INT TSTP` at the start (ignore, do not run handler). This prevents nested
    re-entry while the handler does its (interruptible) state work.
  - Order of operations in on_tstp, all BEFORE `kill -STOP $$`:
    1. `cka_sim::state::set_pause`
    2. `cka_sim::timer::stop`  (kill + wait the background redraw_loop — no orphan)
    3. `cka_sim::state::save`  (now safe — INT/TSTP are ignored)
    4. restore terminal: `stty "$CKA_SIM_EXAM_STTY_SAVED" 2>/dev/null || true` (see defect 4)
    5. re-arm the real handlers: `trap 'cka_sim::exam::on_tstp' TSTP` and
       `trap 'cka_sim::exam::on_int' INT` — re-armed BEFORE the stop so a Ctrl-Z while
       stopped is caught on the next CONT, not lost to default disposition.
    6. clear the re-entry guard: `CKA_SIM_EXAM_IN_SIGHANDLER=0`
    7. `kill -STOP $$`  — this is the LAST line of on_tstp.
  - Move ALL resume work into on_cont ONLY (on_tstp must not do post-resume work). on_cont
    runs on SIGCONT after `fg`. on_cont must: `cka_sim::state::add_pause_delta`;
    `cka_sim::state::save`; re-save the terminal mode
    (`CKA_SIM_EXAM_STTY_SAVED=$(stty -g 2>/dev/null || true)`); `cka_sim::timer::spawn
    "$CKA_SIM_EXAM_DEADLINE_TS"`; print the "✓ Resumed." line. Guard on_cont against running
    when not actually paused: if `CKA_SIM_EXAM_PAUSED_AT` is 0, just return (CONT can fire in
    other situations). This makes on_tstp + on_cont a single non-racing pause→resume path:
    on_tstp owns pause + stop-timer, on_cont owns delta + respawn-timer. Neither spawns or
    stops the timer that the other already handled.

  DEFECT 4 — stty save/restore + gate the background timer:
  - In exam.sh add `declare -g CKA_SIM_EXAM_STTY_SAVED=""`. In start_new (after the manifest
    loads, before `cka_sim::timer::spawn`/`question_loop`, around :362-365) and in resume
    (before its timer spawn, ~:399) save the terminal mode:
    `CKA_SIM_EXAM_STTY_SAVED=$(stty -g 2>/dev/null || true)`.
  - on_tstp restores it before STOP (step 4 above); on_cont re-saves it after resume.
  - Add a timer gate so background `tput` does not collide with a foreground `read` or a
    paused shell. In exam-timer.sh: introduce a gate file path
    `CKA_SIM_TIMER_GATE="${CKA_SIM_TIMER_GATE:-${TMPDIR:-/tmp}/cka-sim-timer-gate.$$}"`
    (declare -g, exported). In redraw_loop, at the top of each iteration, if the gate file
    EXISTS, skip the `tput sc/cup/el/...` drawing block for that iteration (still sleep, still
    recompute remaining) — i.e. stay silent while gated. Add two helpers:
    `cka_sim::timer::gate_on() { : > "$CKA_SIM_TIMER_GATE"; }` and
    `cka_sim::timer::gate_off() { rm -f "$CKA_SIM_TIMER_GATE"; }`. In timer::stop also
    `rm -f "$CKA_SIM_TIMER_GATE"`.
  - In exam.sh question_loop: call `cka_sim::timer::gate_on` immediately before the
    `read -r action` retry loop and `cka_sim::timer::gate_off` immediately after a successful
    read returns — so the timer never draws while `read` owns the terminal. Do the same
    around the confirm_submit read. on_tstp's `timer::stop` already removes the gate file.
  - Net effect: background timer draws between questions only; during a `read` or a pause it
    is silent; terminal mode is always restored on resume.
  </action>

    <automated>bash -n cka-sim/lib/cmd/exam.sh && bash -n cka-sim/lib/exam-timer.sh && grep -c 'CKA_SIM_EXAM_IN_SIGHANDLER' cka-sim/lib/cmd/exam.sh && grep -nE 'stty -g|gate_on|gate_off|CKA_SIM_TIMER_GATE' cka-sim/lib/cmd/exam.sh cka-sim/lib/exam-timer.sh | grep -v '^[^:]*:[0-9]*:#'</automated>
  </verify>
  <done>
  Both files parse clean. on_tstp is guarded against re-entry, ignores INT/TSTP for its
  duration, stops the timer + saves state + restores stty + re-arms handlers all BEFORE
  `kill -STOP $$`, and `kill -STOP $$` is its last statement. on_cont is the sole resume path
  (delta + timer respawn + stty re-save) and no-ops when not paused. exam.sh saves stty mode
  on exam start, restores on pause, re-saves on resume. exam-timer.sh has a gate file checked
  in redraw_loop with gate_on/gate_off helpers, and question_loop/confirm_submit gate the
  timer around every `read`.
