<interfaces>
**exam-timer.sh** — Background subshell that renders `⏱ HH:MM:SS remaining` at the bottom row of the terminal every second. Spawned by exam.sh after session init or resume. Killed by exam.sh on pause, exit, or exam end.

**exam.sh** — Main orchestrator. Entry points:
- `cka-sim exam <blueprint>` — start new exam session
- `cka-sim exam --resume <ts>` — resume interrupted session

Sources all lib/exam-*.sh helpers. Owns signal traps. Drives the question loop. Calls batch_grade at end. Calls report::render to produce the Markdown report.

**Signal contract (RUN-06):**
- INT (Ctrl-C): flag current question, persist state, return to prompt. Does NOT exit.
- TSTP (Ctrl-Z): persist state with paused_at, kill timer, `kill -STOP $$`. On fg (CONT): shift deadline, respawn timer.
- EXIT: persist state, kill timer, run reset.sh for all set-up questions.

**Question loop (D-11..D-14):**
- For each question in blueprint order: reset.sh → setup.sh → display question.md → wait for action
- Actions: Enter/n = advance, f = flag, s = skip, p = prev (navigate back), q = end exam early
- Navigation is non-linear: candidate can go back to flagged questions
- End-of-exam: confirm_submit screen → batch_grade → report::render

**Resume (D-08, D-09):**
- Re-read session.json, recompute remaining = deadline_ts - now()
- If expired: batch-grade completed questions, render partial report
- If valid: reset+setup current question, respawn timer, re-enter loop
</interfaces>

<threat_model>
### T-7-05: Signal race between INT and state save
- **Threat:** Ctrl-C fires during `cka_sim::state::save` (between jq write and mv), corrupting state.
- **Impact:** medium — session lost.
- **Mitigation:** Atomic mktemp+mv means the save is a single `mv` syscall — effectively atomic. INT during jq write only corrupts the temp file, which is validated by `jq empty` before mv. If jq empty fails, the temp is deleted and the original state file is untouched.
- **Verification:** `tests/exam/state_atomic_write.sh` (from plan 07-01) + `tests/exam/signal_handlers.sh` INT test.

### T-7-06: Timer subshell outlives parent
- **Threat:** Parent crashes without running EXIT trap (e.g., SIGKILL). Timer subshell keeps writing to terminal.
- **Impact:** low — cosmetic annoyance, terminal reset fixes it.
- **Mitigation:** Timer subshell checks `kill -0 $PPID` every tick; if parent is gone, exit. Also: timer uses `$$` (parent PID) stored at spawn time, not `$PPID` (which may differ in nested shells).
- **Verification:** Manual observation. Not worth an automated test (SIGKILL is untrappable by definition).

### T-7-07: Batch grade runs grade.sh against stale namespace
- **Threat:** Between question setup and end-of-exam batch grade, the candidate accidentally deletes the lab namespace. Grade.sh fails with "namespace not found."
- **Impact:** medium — that question scores 0 with no trap diagnostics.
- **Mitigation:** Each grade.sh already handles missing resources gracefully (assertions fail → traps fire → exit 1 with SCORE: 0/N). The report shows the question as failed with relevant traps. This is acceptable behavior — the candidate broke their own lab.
- **Verification:** Documented as known behavior in blueprint README.
</threat_model>
