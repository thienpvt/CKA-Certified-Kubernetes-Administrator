**Core Value:** A candidate can take a 2-hour timed mock exam against their own cluster and get an honest, trap-aware score telling them exactly which CKA domains and which classes of mistake they need to drill before sitting the real exam.

## v1.0.3 Requirements

Five fix-bug requirements. No new features. Each maps to a forensic finding from v1.0.2 close-out (`.planning/forensics/FORENSIC-v102.md`) or v1.0.2 carry-over.

### Drill UX

- [ ] **DRILL-NS-01**: When a candidate runs `cka-sim drill <pack> [<n>]` and the question prompt is rendered, every `${CKA_SIM_LAB_NS}` literal in `question.md` is substituted with the resolved namespace (e.g. `cka-sim-storage-01`) before display. Mirror exam-mode quick task `260517-hvo` (commit on `cka-sim/lib/cmd/exam.sh`). Single-point harness fix in `cka-sim/lib/cmd/drill.sh:321` (`cat "$CKA_SIM_QUESTION_DIR/question.md"` → envsubst-aware render). 20+ `question.md` files keep their `${CKA_SIM_LAB_NS}` literals; the fix is in the renderer, not the prose.

### Backlog Carry-Over

- [ ] **BLG-06**: Per-finding shellcheck/yamllint triage. Plan 17-05 (`a77712a`) scaffolded `continue-on-error: true` on the GHA `validate-local` job so red findings don't block CI; per-finding fixes deferred to v1.0.3. Walk each finding (shellcheck warning OR yamllint warning), classify (fix in code / relax lint config with rationale / out of scope), and remove `continue-on-error: true` once the corpus is clean.
- [ ] **BLG-07**: GHA bash-tests environmental reds on `ubuntu-latest`. 9 unit-test cases red on GHA Ubuntu runner but green on Docker Ubuntu 22.04 + 24.04 + Windows MSYS + local kind. Symptom: `expected 1 got 0` on `cka_sim::baseline::is_candidate_modified` unchanged-baseline branch. Cascades through 4 `traps_*` cases (`default-sa-used`, `hostpath-pv-without-nodeaffinity`, `missing-dns-egress`, `ownership_gate`) via the ownership-gate code path. Root-cause the runner-specific delta (jq version, locale, `set -u` interaction).

### Audit Closure

- [ ] **AUDIT-W&S06**: `workloads-scheduling/06-static-pod` `setup.sh` fails on the v1.0.1 lab cluster during `cka-sim audit` (Step 2 of v1.0.2 close-out UAT, ns=cka-sim-audit-workloads-scheduling-06-static-pod). Not a kind-skip — real-cluster setup-drift. Identify the failure (likely `/etc/kubernetes/manifests/` permissions or worker-node SSH issue) and either fix `setup.sh` to succeed on a 1+2 kubeadm cluster, OR document the unsupported-on-lab-cluster constraint with the same exclusion shape Phase 17 added for kind.
- [ ] **LINT-01**: The symptom-diff regression test (`cka-sim/tests/cases/symptom-diff-regression.sh`) silently passes when comparison fails — `lint-question-symptom.sh` reports `✓ symptom diff: 1 question(s) passed` while `cka-sim/lib/symptom-diff.sh:94` emits `Bad file descriptor`. The regression test deliberately mutates `expected-symptom.yaml` to assert the lint catches drift; it doesn't, so Phase 15's quality gate is silently broken on subsequent milestones. Root-cause the line 94 fd 3 reference, fix the swallowed-error path so lint exits 1 on real drift, and re-verify the regression test catches the mutation.
