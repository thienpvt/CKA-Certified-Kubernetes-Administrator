<!-- Helper logic from cka-sim/lib/baseline.sh:264-275 (rv-fallback path) -->
- Line 264-269: `current_rv` is captured via `kubectl get ... -o jsonpath='{.metadata.resourceVersion}' 2>/dev/null`.
- Line 271-273: `[[ "$current_rv" != "$baseline_rv" ]]` returns 0 (modified). Otherwise falls through.
- Line 275: returns 1 (unchanged).
- BUG: when `current_rv=""` (jq returns empty) and `baseline_rv="100"`, the comparison `[[ "" != "100" ]]` is true → modified. This is the wrong default for an unreadable current state.

<!-- Stub jsonpath translator from cka-sim/tests/bin/kubectl:84-94 -->
- Line 84-87: sed translates kubectl jsonpath to jq expression.
- Line 91-93: BUG-M11 fix uses `as $v | $v // ""` shape inside `jq -r`. This shape can emit unexpected output on certain jq versions when the source value is a JSON string vs integer (the fixture has `"resourceVersion": "100"` — a string).
- Line 93: `tr '\n' ' ' | sed 's/[[:space:]]*$//'` strips trailing whitespace, but pipeline behavior under `set -euo pipefail` differs across shell + jq versions.

<!-- Conservative fix shape per CONTEXT D-BLG-07 -->
- Treat empty `current_rv` (or empty `current_gen` in the gen-first branch where it falls through) as "unchanged" rather than "modified".
- Rationale: an unreadable current state is an environment problem, not evidence the candidate modified the resource. Defaulting to "unchanged" prevents false positives in trap detectors.
- Same defensive default the back-compat path uses (line 207-209: returns 0 when CKA_SIM_BASELINE_PATH is missing — but that's "skip the gate entirely"; here we want "passed the gate as unchanged").
