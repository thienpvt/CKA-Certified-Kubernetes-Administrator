From cka-sim/lib/symptom-diff.sh (lines 91-96 — bug site):
- `_emit_row()` is the one fd-3 emitter; called 8+ times from `run_one` (ERROR/MISSING/PASS/FAIL paths).
- Current body uses a brace group with `>&3 2>/dev/null || true`.
- Bug: bash evaluates `>&3` BEFORE the inner stderr redirect; when fd 3 is NOT open (lint mode), bash emits `<file>: line 94: 3: Bad file descriptor` directly on the script's stderr, and the inner `2>/dev/null` does NOT catch it. The `|| true` masks the redirect-failure exit but the stderr leak fires unconditionally and the printf never runs.

From cka-sim/lib/cmd/audit.sh (lines 253-260 — audit-mode caller; fd 3 IS open):
- `cka_sim::symptom_diff::run_one ... "audit" 3>"$tsv_tmp"` opens fd 3 inside a brace group.
- `_emit_row` calls inside `run_one` write TSV rows to that fd; rendered later by `cka_sim::audit::_render_question`.

From cka-sim/scripts/lint-question-symptom.sh (line 62 — lint-mode caller; fd 3 NOT open):
- `if ! cka_sim::symptom_diff::run_one "$yaml_file" "$q_dir" "$pack" "$q_name" "lint"; then` — no fd-3 redirect.
- Lint mode reads `err "..."` → stderr (human-readable citations); `_emit_row` rows are an audit-mode-only side channel by design.

From cka-sim/tests/cases/symptom-diff-regression.sh (Phase 15 plan 07 — the canary):
- Mutates `cka-sim/packs/storage/01-pvc-binding/expected-symptom.yaml`: `status.phase: Pending` → `status.phase: Bound`.
- Runs `bash cka-sim/scripts/lint-question-symptom.sh storage/01-pvc-binding` twice (once for output capture, once for rc).
- Asserts: rc != 0 AND captured output contains `expected 'Bound', got 'Pending'`.
- Cluster-info gated: skips with rc=0 if no live cluster reachable.
- Restores the YAML on EXIT trap.

Key insight from `cka-sim/current-tests/step1-results.txt` lines 632-637: the lint already EMITS the `err "$yaml_file:LINE: PVC/data status.phase expected 'Bound', got 'Pending'"` at symptom-diff.sh:250 → `local_fail=1` is set → `run_one` returns 1 → lint script's `if ! ... ; then errors=$((errors + 1))` fires → final `if (( errors > 0 ))` exits 1. So the regression test SHOULD catch the mutation already via the err-line + non-zero rc path. The reason it doesn't (per step1-results.txt:625-637) is that the FAIL path is never reached on certain executions — repeated `Bad file descriptor` failures from `_emit_row` calls earlier in `run_one` (the ERROR-row at line 115/126/162 BEFORE the FAIL row at line 251) interact with `set -uo pipefail` such that diagnostic state is masked. After the fd-3 fix, `_emit_row` becomes a quiet no-op in lint mode, the function reaches the FAIL emission, `local_fail=1` is set, rc=1, lint exits 1, regression test passes.
