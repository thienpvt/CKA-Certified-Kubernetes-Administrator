    ```
    # storage/05-wait-for-first-consumer

    **Domain:** Storage  |  **Estimated time:** 7 minutes

    A PVC `q05-claim` in `${CKA_SIM_LAB_NS}` is stuck `Pending`. Events show `waiting for first consumer to be created before binding`. The StorageClass `q05-wffc` is intentional — you must work with it, not around it.

    ## Tasks

    1. Inspect `q05-claim` and the StorageClass `q05-wffc` to understand why binding is deferred.
    2. Get the PVC to `Bound` without modifying the StorageClass or the PVC itself.

    ## Constraints

    - Do NOT change `q05-claim.spec.storageClassName`.
    - Do NOT delete the StorageClass.

    ## Verify yourself

    ```
    kubectl get pvc q05-claim -n ${CKA_SIM_LAB_NS}    # STATUS should be Bound
    ```
    ```

    **setup.sh** (chmod +x):
    ```
    #!/bin/bash
    set -euo pipefail
    : "${CKA_SIM_LAB_NS:?CKA_SIM_LAB_NS must be set}"
    : "${CKA_SIM_ROOT:?CKA_SIM_ROOT must be set}"
    source "$CKA_SIM_ROOT/lib/setup.sh"

    cka_sim::setup::ensure_lab_ns "$CKA_SIM_LAB_NS" storage storage-wait-for-first-consumer
    cka_sim::setup::wait_for_ns_active "$CKA_SIM_LAB_NS" storage storage-wait-for-first-consumer 120

    # StorageClass with volumeBindingMode WFFC, no dynamic provisioner.
    kubectl apply -f - <<'EOF'
    apiVersion: storage.k8s.io/v1
    kind: StorageClass
    metadata:
      name: q05-wffc
    provisioner: kubernetes.io/no-provisioner
    volumeBindingMode: WaitForFirstConsumer
    reclaimPolicy: Retain
    EOF

    # Manual hostPath PV pinned to a worker node (any node with the standard hostname label).
    cka_sim::setup::seed_pv_hostpath q05-wffc-pv 1Gi ReadWriteOnce Retain /tmp/q05-wffc-pv kubernetes.io/hostname
    # Patch: tag the PV with the StorageClass name so the binder matches it.
    kubectl patch pv q05-wffc-pv --type=json -p='[{"op":"add","path":"/spec/storageClassName","value":"q05-wffc"}]' || true

    # PVC q05-claim -- stays Pending until a consumer Pod is scheduled.
    kubectl apply -f - <<EOF
    apiVersion: v1
    kind: PersistentVolumeClaim
    metadata:
      name: q05-claim
      namespace: ${CKA_SIM_LAB_NS}
    spec:
      accessModes: [ReadWriteOnce]
      resources: {requests: {storage: 500Mi}}
      storageClassName: q05-wffc
    EOF
    ```

    **grade.sh** (chmod +x):
    ```
    #!/bin/bash
    set -uo pipefail
    : "${CKA_SIM_LAB_NS:?}"
    : "${CKA_SIM_ROOT:?}"
    source "$CKA_SIM_ROOT/lib/grade.sh"
    source "$CKA_SIM_ROOT/lib/traps.sh"

    kubectl wait --for=condition=Ready pod/q05-consumer -n "$CKA_SIM_LAB_NS" --timeout=60s 2>/dev/null || true

    cka_sim::grade::assert_pod_ready "$CKA_SIM_LAB_NS" q05-consumer
    cka_sim::grade::assert_pvc_bound "$CKA_SIM_LAB_NS" q05-claim
    cka_sim::grade::assert_field_eq pod q05-consumer \
      '{.spec.volumes[0].persistentVolumeClaim.claimName}' \
      'q05-claim' \
      -n "$CKA_SIM_LAB_NS"

    # Trap: PVC still Pending + no pod consuming it -> pvc-pending-wffc-unscheduled-consumer
    pvc_phase=$(kubectl get pvc q05-claim -n "$CKA_SIM_LAB_NS" -o jsonpath='{.status.phase}' 2>/dev/null || echo "")
    pod_exists=$(kubectl get pod q05-consumer -n "$CKA_SIM_LAB_NS" -o name 2>/dev/null || echo "")
    if [[ "$pvc_phase" == "Pending" && -z "$pod_exists" ]]; then
      cka_sim::grade::record_trap pvc-pending-wffc-unscheduled-consumer
    fi

    # Trap: default SA used for the consumer pod
    if [[ -n "$pod_exists" ]]; then
      tid=$(cka_sim::trap::detect_default_sa_used "$CKA_SIM_LAB_NS" q05-consumer)
      [[ -n "$tid" ]] && cka_sim::grade::record_trap "$tid"
    fi

    cka_sim::grade::emit_result
    ```
