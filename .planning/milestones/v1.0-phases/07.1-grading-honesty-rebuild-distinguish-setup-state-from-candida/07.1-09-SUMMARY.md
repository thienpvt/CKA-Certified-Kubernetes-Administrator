## Decisions Made

See `key-decisions` frontmatter. Key reasoning recap:

- **Env-overridable retry loops (Q04, Q08):** The kubectl-stub fixture model can't simulate metrics-server latency or rolling-update settle time. Without env knobs, Q04 burns 60s and Q08 burns 30s in CI per case. Defaults preserve production cluster behaviour.
- **Q05 trap gating:** Existing `daemonset-missing-control-plane-toleration` trap fires on ANY empty toleration, including when the DS doesn't exist. Added existence pre-check so empty submission doesn't trigger a spurious trap that contradicts the Wave 2 "0 traps on empty" must-have for assertion-only Qs.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing Critical] Q04 + Q08 retry loops not testable in kubectl-stub model**

- **Found during:** Task 2 (capture fixtures)
- **Issue:** Q04 has a 12-iter × 5s sleep loop waiting for metrics-server scrape; Q08 has a 15-iter × 2s sleep loop polling Running pod count. In kubectl-stub mode the stub returns deterministic state instantly, but bash `sleep` is real. CI would burn 60s + 30s per case.
- **Fix:** Added env-overridable retry knobs (`CKA_SIM_GRADE_TOP_RETRIES`, `CKA_SIM_GRADE_TOP_SLEEP`, `CKA_SIM_GRADE_POLL_RETRIES`, `CKA_SIM_GRADE_POLL_SLEEP`); defaults preserve production behaviour (12 × 5s and 15 × 2s respectively).
- **Files modified:** cka-sim/packs/workloads-scheduling/04-hpa-metrics-server/grade.sh, cka-sim/packs/workloads-scheduling/08-nodeselector-affinity-taints/grade.sh
- **Verification:** Test cases set retries=1 + sleep=0; full test.sh completes in seconds; production behaviour unchanged.
- **Committed in:** 4ea518f (Task 2 commit)
- **Audit-escape annotation:** Both grade.sh files carry `# Phase 07.1 D-22 audit-escape: ...` comments; documented in 07.1-09-AUDIT-ESCAPE.md.

**2. [Rule 1 - Bug] Q05 daemonset-missing-cp-toleration trap fires on empty submission**

- **Found during:** Task 2 (fixture dry-run)
- **Issue:** The Q05 grade.sh records the `daemonset-missing-control-plane-toleration` trap whenever the CP toleration key is empty. On empty submission the DaemonSet doesn't exist, so the kubectl jsonpath returns empty → trap fires spuriously, contradicting the Wave 2 "0 traps on empty" contract.
- **Fix:** Gated the trap firing on a preceding `kubectl get daemonset q05-node-agent -o name` existence check. If DS doesn't exist, skip the trap check entirely.
- **Files modified:** cka-sim/packs/workloads-scheduling/05-daemonset/grade.sh
- **Verification:** Q05 test asserts `0 traps recorded` on empty submission; passes.
- **Committed in:** 4ea518f (Task 2 commit)

## Issues Encountered

- **Bash `$(...)` strips trailing newlines, breaking `wc -l` for stub-emitted multi-line stdout.** Realized while dry-running Q05 fixture. `kubectl get nodes --no-headers | wc -l` counts only internal newlines because the trailing `\n` in the JSON value is stripped during stub stdout capture. Workaround: model fixture stdout as "N+1 nodes" when grader expects `wc -l = N`. Documented in test case comments for future fixture authors.
- **jq on Windows emits `\r\n` for JSON `\n` escape.** Visible in `od -c` output of stub stdout. Doesn't break test logic (bash treats `\r` as part of token, `wc -l` still counts only `\n`s) but worth noting for cross-platform reproducibility.
