## D-22 Audit Escapes

### Q06 — Static Pod (on-node manifest)

- **Annotation in grade.sh:** `# Phase 07.1 D-22 audit-escape: on-node static-pod check; kubectl-stub fixture model partially applies (mirror pod is namespaced)`
- **Reason:** Q06 verifies the kubelet has mirrored a candidate-authored
  `/etc/kubernetes/manifests/q06-static-nginx.yaml` on `node-01`. The mirror
  pod itself is namespaced in `default`, so `kubectl get pod` returns it and
  the grade.sh assertions remain in the kubectl-stub fixture model. However:
  - The candidate authoring step (drop yaml on node-01) is an **on-node**
    action — the fixture cannot simulate "manifest absent on node-01" vs
    "manifest present" beyond the `kubectl get pod` outcome it produces.
  - The static-pod author check (`assert_resource_candidate_authored`) is
    correct in that the mirror pod is never in setup baseline (setup only
    does SSH preflight + ns create), so post-setup fixtures must produce
    rc=1 for `kubectl get pod ... -o name`.
- **Mitigation:** Plan 13 should track adding an on-node trace harness for
  Q06 to verify the candidate's `sudo rm /etc/kubernetes/manifests/...` reset
  semantics. The current kubectl-stub regression captures the mirror-pod
  visibility contract, which is what the grader checks.
