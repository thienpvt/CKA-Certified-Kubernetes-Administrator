    ```bash
    # Phase 07.1 — multi-resource fixture dispatch via .fixtures.json manifest.
    # If $CKA_SIM_TEST_CURRENT/.fixtures.json exists, look up the canonicalized argv as a key.
    # On match: emit stdout from manifest entry, exit with rc from entry.
    # On miss: fall through to legacy single-fixture path below.
    fixtures_manifest="$CKA_SIM_TEST_FIXTURES_DIR/$CKA_SIM_TEST_CURRENT/.fixtures.json"
    if [[ -r "$fixtures_manifest" ]]; then
      argv_key="$*"
      entry=$(jq -r --arg k "$argv_key" '.[$k] // empty' "$fixtures_manifest" 2>/dev/null)
      if [[ -n "$entry" ]]; then
        out=$(jq -r --arg k "$argv_key" '.[$k].stdout // ""' "$fixtures_manifest")
        rc=$(jq -r --arg k "$argv_key" '.[$k].rc // 0' "$fixtures_manifest")
        [[ -n "$out" ]] && printf '%s' "$out"
        exit "$rc"
      fi
      # Manifest exists but no entry matches — emit canonicalized argv to stderr
      # so the test author can see what key to add. Then continue to legacy path.
      printf 'kubectl-stub: no manifest entry for argv: %s\n' "$argv_key" >&2
    fi
    # Legacy single-fixture path (Phase 2-7 existing tests rely on this).
    # ... (existing code unchanged below) ...
    ```

    Place this block AFTER the existing `: "${CKA_SIM_TEST_CURRENT:?...}"` guard but BEFORE the `verb="${1:-}"` line. The legacy verb-dispatching code stays unchanged below it. Existing 70+ tests work unchanged because they don't supply a .fixtures.json manifest — the if-branch is bypassed.

    expected_max=<EXPECTED_MAX>  # 1 for workloads-scheduling/02 (delta), 3 for storage/02, 6 for s-n/06, 1 for c-a/04
