## 1. Implement `cka-sim/lib/preflight.sh`
Shared helpers sourced by `bootstrap.sh` and `doctor.sh`:

- `cka_sim::preflight::check_binaries <name...>` — for each, `command -v` check; returns list of missing. Never prints unless verbose.
- `cka_sim::preflight::check_cluster_nodes` — runs `kubectl get nodes -o json` once, asserts ≥3 Ready, ≥1 control-plane (label `node-role.kubernetes.io/control-plane`), ≥2 workers. Echoes counts on success. Non-zero exit with reason on failure.
- `cka_sim::preflight::get_worker_names` — echoes worker hostnames, one per line, from `kubectl get nodes -o jsonpath='{range .items[?(!@.metadata.labels.node-role\.kubernetes\.io/control-plane)]}{.metadata.name}{"\n"}{end}'`.
- `cka_sim::preflight::get_node_ip <name>` — echoes InternalIP for given node via jsonpath.
- `cka_sim::preflight::check_kubeconfig` — `test -r` on `${KUBECONFIG:-/etc/kubernetes/admin.conf}` then fall back to `~/.kube/config`. Sets `KUBECONFIG` env if resolved.
- `cka_sim::preflight::check_ssh_batchmode <node>` — `ssh -o BatchMode=yes -o ConnectTimeout=3 <node> hostname` ; echoes `ok` or `fail:<reason>`.
- `cka_sim::preflight::check_state_dirs` — `test -d` on each of `~/.cka-sim/{sessions,history,reports,logs}`.
- `cka_sim::preflight::ensure_state_dirs` — `mkdir -p` the four subdirs.

## 2. Implement `cka-sim/lib/cmd/bootstrap.sh`
Ordered steps, each idempotent, each printing a one-line `info`/`ok` status. On any failure, `die` with actionable fix.

1. **Header:** `header "cka-sim bootstrap"`
2. **Preflight check:** `check_kubeconfig` → if fails, `die "kubeconfig not readable at ~/.kube/config or /etc/kubernetes/admin.conf — copy admin.conf to ~/.kube/config and re-run"`
3. **Cluster nodes check:** `check_cluster_nodes` → die with node-count error if <3 Ready
4. **Install jq if missing:** if `! command -v jq`; prompt "jq not installed. Install via sudo apt-get install -y jq? [y/N]"; if y: run `sudo apt-get install -y jq`; if non-zero exit or user said N: `die "jq is required — install manually with 'sudo apt-get install -y jq' and re-run"`
5. **State dirs:** `ensure_state_dirs` then `ok "state dirs at ~/.cka-sim/"`
6. **Shell profile env block:** use `_write_bashrc_block` helper (see below) to install sentinel-fenced block in `~/.bashrc`:
   ```
   # === cka-sim BEGIN (managed by cka-sim bootstrap; do not edit inside this block) ===
   export ETCDCTL_API=3
   export CONTAINER_RUNTIME_ENDPOINT=unix:///run/containerd/containerd.sock
   # === cka-sim END ===
   ```
   Strategy: `sed -i '/# === cka-sim BEGIN/,/# === cka-sim END/d'` then append fresh block. Creates `~/.bashrc` if missing.
7. **SSH key:** if `~/.ssh/cka_sim_ed25519` missing, run `ssh-keygen -t ed25519 -N '' -f ~/.ssh/cka_sim_ed25519 -C "cka-sim@$(hostname)"`. `chmod 700 ~/.ssh` if needed.
8. **SSH config stanzas:** use `_write_sshconfig_block` helper: sentinel-fenced block in `~/.ssh/config`:
   ```
   # === cka-sim BEGIN (managed by cka-sim bootstrap; do not edit inside this block) ===
   <for each worker $name, $ip from get_worker_names/get_node_ip>
   Host $name
     HostName $ip
     User <current user>
     IdentityFile ~/.ssh/cka_sim_ed25519
     StrictHostKeyChecking accept-new
     UserKnownHostsFile ~/.ssh/known_hosts
     ControlMaster auto
     ControlPath ~/.ssh/cm-%r@%h:%p
     ControlPersist 10m
   # === cka-sim END ===
   ```
   `chmod 600 ~/.ssh/config`.
9. **Distribute pubkey:** for each worker, check `ssh -o BatchMode=yes $name true`; if fails, run `ssh-copy-id -i ~/.ssh/cka_sim_ed25519.pub -o StrictHostKeyChecking=accept-new $USER@<ip>` (may prompt for password once — unavoidable without pre-existing SSH). Re-check BatchMode.
10. **Install symlink:** if `/usr/local/bin/cka-sim` doesn't exist OR points elsewhere, prompt `"Install /usr/local/bin/cka-sim → $CKA_SIM_ROOT/bin/cka-sim? (needs sudo) [y/N]"`; on y: `sudo ln -sf "$CKA_SIM_ROOT/bin/cka-sim" /usr/local/bin/cka-sim`. Idempotent via `ln -sf`.
11. **Final check:** print next step: `"Run 'cka-sim doctor' to verify the cluster is ready."` Exit 0.

## 3. Implement `cka-sim/lib/cmd/doctor.sh`
Runs the preflight checks, one by one, printing a `✓` line on success and a `✗ <actionable fix>` on failure. Collects failures; exits 1 if any failed, 0 if all green.

Checks in order:
1. Required binaries: `kubectl jq etcdctl crictl ssh ssh-keygen`
2. Kubeconfig readable
3. `kubectl get --raw /livez` returns ok
4. Cluster topology: ≥3 Ready nodes, ≥1 control-plane, ≥2 workers
5. For each worker: `ssh -o BatchMode=yes -o ConnectTimeout=3 <name> hostname` succeeds
6. `~/.cka-sim/` state dirs present
7. `~/.ssh/cka_sim_ed25519` exists
8. sentinel block present in `~/.bashrc` AND `ETCDCTL_API=3` in current env OR sourceable

On fail, each check prints its own fix command (e.g. `"✗ node-01 unreachable via ssh BatchMode — run 'cka-sim bootstrap' to (re-)distribute the key"`).

## 4. Small helpers: sentinel-block write
Shared function in `cka-sim/lib/fileblock.sh` (new tiny file):
```bash
cka_sim::fileblock::write() {
  local file="$1" begin_marker="$2" end_marker="$3" content="$4"
  touch "$file"
  # Remove any existing block
  if grep -qF "$begin_marker" "$file"; then
    sed -i "/$begin_marker/,/$end_marker/d" "$file"
  fi
  # Append fresh
  {
    printf '%s\n' "$begin_marker"
    printf '%s\n' "$content"
    printf '%s\n' "$end_marker"
  } >> "$file"
}
```
bootstrap.sh sources this from `$CKA_SIM_ROOT/lib/fileblock.sh`.
