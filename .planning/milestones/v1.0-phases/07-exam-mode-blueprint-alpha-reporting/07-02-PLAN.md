<interfaces>
**Input:** Path to a fully-populated session.json (schema v1 from exam-state.sh). All questions have `grade_raw`, `score`, `max_score`, `traps[]` populated.

**Output:** Markdown file at the specified output path. Layout per CONTEXT D-21:
```markdown
# CKA Exam Report — <ts>

**Blueprint:** <id>
**Started:** <iso8601>  **Completed:** <iso8601>  **Duration:** <hh:mm:ss>
**Total Score: <N>/100  (<pass|fail> vs 66% pass mark)**

## Per-Domain Breakdown (weakest first)
| Domain | Score | Percentage | Blueprint Weight |
...

## Top 5 Traps Hit
| # | Trap ID | Count | Description |
...

## Suggested Next Drills
...

## Question-by-Question Detail
| # | Domain | Question | Score | Status | Traps |
...
```

**Dependencies:** Sources `lib/exam-state.sh` (for `cka_sim::state::now` and session path helpers). Reads `$CKA_SIM_ROOT/traps/catalog.yaml` for trap descriptions (grep-based lookup, not full parse).

**Score computation (D-23):** For each domain d with weight w_d: domain_score_d = sum(q.score) for questions in d; domain_max_d = sum(q.max_score) for questions in d; domain_pct_d = domain_score_d / domain_max_d * 100. Total = sum(domain_pct_d * w_d / 100) for all domains. Pass if total >= 66.
</interfaces>

<threat_model>
### T-7-03: Report score computation off-by-one or rounding error
- **Threat:** Weighted score computation produces a different total than expected due to integer division truncation or floating-point rounding in jq/awk.
- **Impact:** medium — candidate sees wrong pass/fail verdict.
- **Mitigation:** Golden-file test with hand-computed expected values catches any arithmetic drift. Use `awk` for floating-point division (not bash integer arithmetic). Round final total with `printf '%.0f'`.
- **Verification:** `tests/exam/report_golden.sh` byte-compares output against expected-report.md.

### T-7-04: Trap description lookup fails silently
- **Threat:** A trap ID in session.json doesn't exist in catalog.yaml, causing grep to return empty. Report shows blank description column.
- **Impact:** low — cosmetic, not functional.
- **Mitigation:** Fallback: if grep returns empty, use the trap ID itself as the description. Golden test includes at least one trap that exists in the mock catalog.
- **Verification:** Golden test covers this path.
</threat_model>
