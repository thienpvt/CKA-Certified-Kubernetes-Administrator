<threat_model>
### T-7-01: JSON state corruption on concurrent write
- **Threat:** Two processes (exam + a stale resume) write session.json simultaneously, producing invalid JSON.
- **Impact:** medium — session lost, candidate must restart exam.
- **Mitigation:** Atomic mktemp+mv ensures each write is all-or-nothing. `jq empty` validation between write and mv catches marshalling bugs. Single-candidate mode (PROJECT.md constraint) means concurrent exams are undefined behavior — documented, not defended.
- **Verification:** `tests/exam/state_atomic_write.sh` injects write failure and asserts original file integrity.

### T-7-02: Blueprint manifest references non-existent question
- **Threat:** A typo in manifest.yaml (wrong pack or slug) causes exam to fail at runtime with "missing file" error mid-exam.
- **Impact:** high — exam crashes after candidate has invested time.
- **Mitigation:** `cka_sim::blueprint::validate` calls `resolve_question` for every entry, verifying all 6 files exist. lint-packs.sh pass H (plan 07-06) enforces this at CI time. Runtime `cka_sim::blueprint::load` also calls resolve_question.
- **Verification:** `tests/exam/blueprint_validate.sh` tests resolution failure case.
</threat_model>
