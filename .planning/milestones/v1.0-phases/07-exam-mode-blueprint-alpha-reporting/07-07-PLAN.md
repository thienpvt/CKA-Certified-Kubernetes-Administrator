Root cause (from debug session — five compounding defects):
1. PRIMARY — `read` not restarted after a trap. exam.sh:209 (and confirm_submit:242) treats a
   trap-interrupted non-zero `read` as EOF and ends the exam.
2. on_tstp re-arms the TSTP trap only AFTER resume and does interruptible `state::save` work →
   a nested signal re-enters trap handling and `kill -STOP $$` may never be reached.
3. on_tstp tail + on_cont both touch timer spawn/stop and pause accounting → races, double-spawned
   (orphaned) redraw_loop.
4. No `stty` save/restore around pause/resume; background timer `tput` collides with foreground
   `read` → terminal left non-echo/non-canonical, next `read` blocks invisibly.
5. setup.sh runs as a foreground child (exam.sh:170) with no `|| true` under `set -e` → an
   interrupted setup returns 130 and kills the whole exam.

Residual root cause after the 07-07 fix (from debug "Re-diagnosis 2026-05-14"):
The 07-07 fix resolved defects 1, 4, 5 and partially 2/3, but kept `kill -STOP $$` (last
line of on_tstp) plus a separate `on_cont` CONT trap. That design cannot survive nested
SIGTSTP:
  (a) STOP SANDWICH — on_tstp clears its re-entry guard and re-arms `trap on_tstp TSTP`
      BEFORE `kill -STOP $$`. A SIGTSTP in that window re-enters on_tstp, stacking a second
      `kill -STOP $$` frame. The guard is useless because it is cleared before the stop.
  (b) on_cont RESETS PAUSED_AT — the first `fg`'s CONT trap runs add_pause_delta (PAUSED_AT=0);
      the second `fg`'s on_cont early-returns → no timer respawn, no "✓ Resumed.", read looks dead.
  (c) TIMER DRAWS WHILE STOPPED — the inner resume respawns the timer, then the outer
      `kill -STOP $$` stops the shell with the timer still running → garbled terminal.

  <action>

  start_new/resume, the timer gate, and the gate_on/gate_off calls all work and stay.

  WHY the current design fails (do not preserve it): on_tstp's last line is `kill -STOP $$`,
  with the re-entry guard cleared and `trap on_tstp TSTP` re-armed BEFORE that line. A second
  SIGTSTP in that few-instruction window re-enters on_tstp → two stacked `kill -STOP $$`
  frames ("stop sandwich"). Plus on_cont runs on the first SIGCONT and zeroes PAUSED_AT via
  add_pause_delta, so the second `fg`'s on_cont early-returns → no timer respawn, no
  "✓ Resumed." The canonical idiom below eliminates BOTH the sandwich and the split-resume.

  REWRITE on_tstp (exam.sh:74-97) to be fully self-contained. After `trap - TSTP; kill -TSTP
  $$`, the kernel performs the DEFAULT stop — no handler runs while stopped, so nothing can
  nest. On SIGCONT (`fg`), execution resumes IN-PLACE at the next line inside the SAME
  handler frame. One `^Z` = one stop = one `fg` = one in-place resume. Exact structure:

    cka_sim::exam::on_tstp() {
      # Re-entry guard — spans the WHOLE handler; effective because no early
      # frame-clearing kill happens before the guard is reset at the end.
      (( CKA_SIM_EXAM_IN_SIGHANDLER )) && return
      CKA_SIM_EXAM_IN_SIGHANDLER=1

      # Block INT/TSTP during the interruptible cleanup work below.
      trap '' INT TSTP

      # --- pause / save / terminal-restore (all before the stop) ---
      cka_sim::state::set_pause
      cka_sim::timer::stop
      cka_sim::state::save
      stty "$CKA_SIM_EXAM_STTY_SAVED" 2>/dev/null || true

      # Restore DEFAULT TSTP disposition, then self-stop. The kernel does the
      # stop with NO handler installed → no nesting, no stop sandwich.
      trap - TSTP
      kill -TSTP $$

      # ===== SIGCONT (fg) resumes execution HERE, same frame =====

      # Re-arm the real handlers now that we are running again.
      trap 'cka_sim::exam::on_tstp' TSTP
      trap 'cka_sim::exam::on_int'  INT

      # --- resume accounting / timer respawn / terminal re-save ---
      cka_sim::state::add_pause_delta
      cka_sim::state::save
      CKA_SIM_EXAM_STTY_SAVED=$(stty -g 2>/dev/null || true)
      cka_sim::timer::spawn "$CKA_SIM_EXAM_DEADLINE_TS"
      printf '\n\033[32m✓ Resumed.\033[0m\n' >&2

      CKA_SIM_EXAM_IN_SIGHANDLER=0
    }

  Notes on the rewrite:
  - INT is intentionally left blocked (`trap '' INT TSTP`) across the cleanup AND across the
    stop — `kill -TSTP $$` returns the moment SIGCONT arrives; INT is re-armed immediately
    after on the resume side. A Ctrl-C delivered while the process is stopped is held by the
    kernel and there is no INT handler to nest; on resume the trap is re-armed before any
    `read` runs again.
  - `kill -TSTP $$` (not `kill -STOP`): TSTP is the job-control stop that pairs with the
    default disposition restored by `trap - TSTP`. `kill -STOP` is uncatchable and would not
    benefit from `trap -`, but more importantly the old code's `kill -STOP` sat AFTER the
    guard was cleared and the handler re-armed — the structural bug. The new ordering puts
    the self-stop while INT/TSTP are still contained and the guard still set.
  - The re-entry guard `CKA_SIM_EXAM_IN_SIGHANDLER` now genuinely spans the whole handler:
    it is set at entry and cleared only at the very end, AFTER the in-place resume completes.
    No `kill -STOP` clears the frame mid-handler anymore.

  RETIRE on_cont (exam.sh:99-109): all resume work now lives inside on_tstp's in-place resume
  section, so on_cont has nothing to do. Reduce `cka_sim::exam::on_cont` to a pure no-op
  safety net:

    cka_sim::exam::on_cont() {
      # Resume logic now lives in on_tstp (in-place after kill -TSTP). This
      # no-op exists only so a stray SIGCONT cannot run resume logic twice.
      return 0
    }

  Justification for keeping a no-op rather than deleting the function and the trap: a stray
  SIGCONT (e.g. job-control quirks, a `kill -CONT` from outside) must NOT run add_pause_delta
  or respawn the timer a second time — that double-resume was part of the original hang. A
  trapped no-op is the explicit, self-documenting guarantee that CONT does nothing; deleting
  the trap would fall back to default CONT disposition which is also fine, but the no-op
  makes the intent unambiguous and costs nothing. Keep the
  `trap 'cka_sim::exam::on_cont' CONT` registration in question_loop (exam.sh:221) as-is,
  now pointing at the no-op. Do NOT delete the registration line.

  DO NOT modify cka-sim/lib/exam-state.sh (set_pause / add_pause_delta / save live there and
  are correct). DO NOT modify exam-timer.sh in this task — its gate and stop/spawn logic are
  fine. DO NOT touch the read loops, setup containment, or stty save points in
  start_new/resume.
  </action>
