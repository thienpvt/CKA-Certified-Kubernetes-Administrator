    Implement the 7 assertion helpers below. All use the `cka_sim::grade::` namespace. Each helper:
    - Increments `CKA_SIM_GRADE_TOTAL` by `${weight}` (default 1) on entry.
    - On pass: increments `CKA_SIM_GRADE_PASSED` by `weight`, appends a one-line message to `CKA_SIM_GRADE_PASSES`, calls bare `ok "..."` (stderr), returns 0.
    - On fail: appends a one-line message to `CKA_SIM_GRADE_FAILS`, calls bare `err "..."` (stderr), returns 1.
    - Never calls `die`. Caller's `set -uo pipefail` keeps grader running.

    Helper signatures + invocation patterns (kubectl jsonpath / verb / endpoint specified per helper):

    a. `cka_sim::grade::assert_resource_exists <kind> <name> [-n <ns>] [<weight>]`
       - Verb: `kubectl get "$kind" "$name" [-n "$ns"] -o name 2>/dev/null`
       - Pass condition: command exits 0 AND stdout non-empty.
       - Fail message: `"resource $kind/$name not found in ns=$ns"`.
       - Pass message: `"resource $kind/$name exists"`.

    b. `cka_sim::grade::assert_field_eq <kind> <name> <jsonpath> <expected> [-n <ns>] [<weight>]`
       - Verb: `kubectl get "$kind" "$name" [-n "$ns"] -o jsonpath="$jsonpath" 2>/dev/null`
       - Pass condition: stdout equals `$expected` exactly.
       - Fail message: `"field $jsonpath on $kind/$name = '<actual>' (expected '<expected>')"`.
       - Pass message: `"field $jsonpath on $kind/$name == '<expected>'"`.

    c. `cka_sim::grade::assert_pod_ready <namespace> <pod-name> [<weight>]`
       - Verb: `kubectl get pod "$pod" -n "$ns" -o jsonpath='{.status.conditions[?(@.type=="Ready")].status}' 2>/dev/null`
       - Pass condition: stdout equals `True`.
       - Fail message: `"pod '$pod' is not Ready (got: '<status>')"`.
       - Pass message: `"pod '$pod' is Ready"`.

    d. `cka_sim::grade::assert_pvc_bound <namespace> <pvc-name> [<weight>]`
       - Verb: `kubectl get pvc "$pvc" -n "$ns" -o jsonpath='{.status.phase}' 2>/dev/null`
       - Pass condition: stdout equals `Bound`.
       - Fail message: `"PVC '$pvc' not Bound (phase='<phase>')"`.
       - Pass message: `"PVC '$pvc' is Bound"`.

    e. `cka_sim::grade::assert_can_i <verb> <resource> [-n <ns>] [--as <user>] [<weight>]`
       - Verb: `kubectl auth can-i "$verb" "$resource" [-n "$ns"] [--as "$user"] 2>/dev/null`
       - Pass condition: command stdout equals `yes` (kubectl prints `yes` / `no`).
       - Fail message: `"can-i $verb $resource (as=$user, ns=$ns) = no"`.
       - Pass message: `"can-i $verb $resource (as=$user, ns=$ns) = yes"`.

    f. `cka_sim::grade::assert_egress_allowed <namespace> <pod-name> <target-host> <target-port> [<weight>]`
       - Verb: `kubectl exec -n "$ns" "$pod" -- timeout 3 sh -c "echo > /dev/tcp/$target_host/$target_port" 2>/dev/null`
       - Pass condition: command exits 0.
       - Fail message: `"egress from pod '$pod' to $target_host:$target_port blocked or unreachable"`.
       - Pass message: `"egress from pod '$pod' to $target_host:$target_port works"`.

    For (a), (b), (e): the helper signature has leading fixed positional args followed by optional flag pairs and an optional trailing weight. Implement the argv parser exactly as follows — do NOT deviate:

    **Parser algorithm (MANDATORY, shared by all helpers that accept flags):**
    1. Shift the fixed leading positional args first (kind+name for `resource_exists`; kind+name+jsonpath+expected for `field_eq`; verb+resource for `can_i`; namespace+pod for `pod_ready`; namespace+pvc for `pvc_bound`; namespace+pod+target-host+target-port for `egress_allowed`; namespace+svc for `endpoints_nonempty`). Each helper knows its fixed arity.
    2. Walk remaining args in a `while (( $# > 0 ))` loop. For each token:
       - If `$1 == "-n"`: require `$2` present (else `die "flag -n requires value"`), set `ns="$2"`, `shift 2`.
       - If `$1 == "--as"` (helper `e` only; other helpers `die "unknown flag: --as"`): require `$2` present, set `user="$2"`, `shift 2`.
       - Else if `$1` matches `^[0-9]+$`: set `weight="$1"`, `shift`. This token MUST be the last one — if more args follow, `die "unexpected argument after weight: $2"`.
       - Else: `die "unexpected argument: $1"` (catches typos, wrong flag order, non-numeric where weight expected).
    3. After the loop, if `weight` was not set, default to `1`.

    **Example traces (verify your parser against these — MUST behave exactly as shown):**
    - `assert_can_i get pods -n cka-sim-test --as alice` → fixed: `verb=get resource=pods`; flags consumed: `ns=cka-sim-test user=alice`; weight defaults to 1. ✓
    - `assert_can_i get pods -n cka-sim-test --as alice 2` → as above plus `weight=2`. ✓
    - `assert_can_i get pods --as alice -n cka-sim-test` → flags in reverse order; parser accepts either order since it loops. ✓
    - `assert_resource_exists Pod my-pod -n cka-sim-test xyz` → die "unexpected argument: xyz" (not numeric, not a flag).
    - `assert_resource_exists Pod my-pod alice` → die "unexpected argument: alice".
