- **Threat:** A later plan (P03-P08) declares a trap-id in `metadata.yaml.traps[]` that is not registered in the catalog. `lint-packs.sh` pass E (`cka_sim::trap::id_exists`) rejects the metadata but the error surface is a per-file grep, not a missing-catalog-entry error — developers lose time debugging.
- **Impact:** low (blocked at lint, not runtime) but time-consuming.
- **Asset:** Phase 6 question authors' iteration speed.
- **Mitigation:** All 11 IDs land in Wave 1 BEFORE any question plan (P03-P08) consumes them; RESEARCH §3 table is the source of truth and every downstream plan references it in its `<read_first>`.
- **Verification:** `bash cka-sim/scripts/lint-traps.sh` reports schema green; P09 VERIFICATION.md criterion 4 re-asserts schema green on the final pack tree.

### T-6-04: RFC-1123 violation in a new trap-id
- **Threat:** A new id contains an uppercase char, underscore, or leading/trailing hyphen. Lint-traps fails; blocks downstream plans.
- **Impact:** low.
- **Asset:** CI pipeline stays unblocked.
- **Mitigation:** All 11 new IDs are locked to the RESEARCH §3 table and are pre-validated RFC-1123 (lowercase + `[a-z0-9-]`, alphanumeric start/end). `lint-traps.sh is_valid_id` regex check catches any deviation.
- **Verification:** acceptance criterion "All 11 new IDs present and RFC-1123" (grep regex `^  - id: $id$` which matches only the locked spellings).

### T-6-04b: Hardcoded baseline count drifts over time
- **Threat:** An earlier draft hardcoded "36 existing + 11 = 47 total". If any upstream plan adds catalog entries before Phase 6 lands, that absolute count silently breaks this plan's acceptance without surfacing the real defect (the delta).
- **Impact:** medium — false-failure on a catalog that is actually correct, or false-pass on a catalog that happens to have 47 entries for the wrong reasons.
- **Asset:** append-only invariant.
- **Mitigation:** Acceptance uses delta checks (`post - pre == 11`) and zero-collision checks, NOT a total row count. Described in the objective.
- **Verification:** acceptance criteria list `pre_count` capture + `git diff --cached` counted-line fallback.
