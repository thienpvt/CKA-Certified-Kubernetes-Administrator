The candidate MUST have executed `kubectl debug node/<worker>` (which auto-labels the created debug pod with `kubectl.kubernetes.io/debug-source`). The leak risk is bounded by reset.sh's unconditional all-namespaces debug-source sweep (T-6-14 mitigation). CI fixture round-trips use the kubectl stub, which replays debug-source-labelled pod data without creating real pods.
