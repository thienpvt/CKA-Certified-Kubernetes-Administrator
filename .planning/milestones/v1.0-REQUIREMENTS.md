# Requirements — Milestone v1.0

**Milestone:** v1.0 CKA Exam Simulator MVP
**Defined:** 2026-05-07
**Status:** Draft (awaiting user approval)

---

## Core Value

A candidate can take a 2-hour timed mock exam against their own 1+2 kubeadm cluster and get an honest, trap-aware score telling them exactly which CKA domains and which classes of mistake they need to drill before sitting the real exam.

---

## v1 Requirements

### Bootstrap — cluster preparation on the existing 1+2 Ubuntu 22.04 cluster

- [ ] **BOOT-01** — `cka-sim bootstrap` runs on the control-plane node and is fully idempotent (safe to re-run any number of times).
- [ ] **BOOT-02** — Bootstrap generates a dedicated ed25519 SSH key (if absent) and distributes it to every worker node so that `ssh node-01` / `ssh node-02` succeeds without a password prompt.
- [ ] **BOOT-03** — Bootstrap writes idempotent `Host node-0N` stanzas (fenced with `# === cka-sim BEGIN/END` sentinel comments) into `~/.ssh/config` with `StrictHostKeyChecking accept-new`, `ControlMaster auto`, discovering node IPs via `kubectl get nodes -o jsonpath=...` (no hard-coded IPs).
- [ ] **BOOT-04** — Bootstrap does **NOT** inject shell aliases, does **NOT** source `scripts/exam-setup.sh`, and does **NOT** touch `~/.vimrc`. Rationale: the candidate practices typing full `kubectl` / `crictl` / `etcdctl` commands to build muscle memory. Any shortcuts are the candidate's opt-in after bootstrap (documentation explains `source scripts/exam-setup.sh` as an opt-in one-liner for exam-day parity, not a default).
- [ ] **BOOT-05** — Bootstrap exports `CONTAINER_RUNTIME_ENDPOINT=unix:///run/containerd/containerd.sock` as an environment variable (not an alias) so `crictl ps` works without a wrapper. Exported to the candidate's shell profile in a sentinel-fenced block (`# === cka-sim BEGIN/END`) so re-running bootstrap never duplicates the export.
- [ ] **BOOT-06** — Bootstrap exports `ETCDCTL_API=3` and verifies `etcdctl version` returns successfully.
- [ ] **BOOT-07** — `cka-sim doctor` preflight command verifies: ≥3 nodes, non-interactive `ssh -o BatchMode=yes` works to every worker, `jq` / `kubectl` / `etcdctl` / `crictl` present, kubeconfig readable. Refuses to start a session if any check fails.

### Runner — CLI, timer, session state

- [ ] **RUN-01** — A single `cka-sim` entry point routes subcommands (`bootstrap`, `drill`, `exam`, `score`, `list`, `doctor`, `version`) via a thin dispatcher; each subcommand is a self-contained bash module under `cka-sim/lib/cmd/`.
- [ ] **RUN-02** — `cka-sim drill <pack> [<n>]` runs one question end-to-end (reset → setup → prompt → grade → report) against a clean lab namespace.
- [ ] **RUN-03** — `cka-sim exam <blueprint>` runs a timed mock: loads the blueprint's ordered question list, starts a visible 2-hour countdown, presents each question, supports flag / skip / return / pause, batch-grades at the end.
- [ ] **RUN-04** — The exam countdown timer updates at least every second on a dedicated status line (using `tput sc` / `tput rc` to preserve the candidate's cursor position) and does not block input.
- [ ] **RUN-05** — Exam sessions persist full state to `~/.cka-sim/sessions/<ts>.json` after every question boundary; `cka-sim exam --resume <ts>` rehydrates cleanly after Ctrl-C, terminal close, or pause.
- [ ] **RUN-06** — Signal handling: Ctrl-C flags the current question and saves state (does NOT kill the exam); Ctrl-Z pauses; normal exit or forced kill both persist state via an `EXIT` trap.

### Runtime contract — the `setup.sh` / `grade.sh` / `reset.sh` triplet

- [ ] **TRIP-01** — Every question ships all three scripts, chmod +x, with a shared authoring template (bash, `set -euo pipefail` on setup/grade, `set -u` only on reset).
- [ ] **TRIP-02** — `setup.sh` is idempotent: running it twice in succession exits 0 with no `AlreadyExists` errors. Verified by a CI lint that actually runs setup twice against an ephemeral cluster fixture (or a seeded dry-run).
- [ ] **TRIP-03** — `setup.sh` creates a per-question lab namespace `cka-sim-<domain>-NN` (distinct from the existing `exercise-NN` convention to prevent collision). Any cluster-scoped object is prefixed with the question ID (e.g. `q07-cr-viewer`).
- [ ] **TRIP-04** — `reset.sh` returns the cluster to a known clean baseline for that question, is idempotent, and uses `--ignore-not-found` to avoid aborting on partial cleanup.
- [ ] **TRIP-05** — The runner always runs `reset.sh` before `setup.sh` for the same question, to defeat cross-session state poison.
- [ ] **TRIP-06** — A setup / reset template enforces sentinel guards for any mutation that could accumulate (append-only files, labels, finalizers).
- [ ] **TRIP-07** — All Kubernetes resource names created by any `setup.sh` (namespaces, pods, cluster-scoped objects, labels, trap IDs, pack IDs, blueprint IDs) MUST conform to RFC 1123 DNS labels: lowercase `[a-z0-9-]` only, start and end with alphanumeric, ≤63 characters. CI lint scans every `setup.sh` and `metadata.yaml` / `manifest.yaml` for non-compliant identifiers and fails the PR. No underscores, no uppercase, no leading/trailing hyphens.

### Grader — trap-aware assertions, not just pass/fail

- [ ] **GRADE-01** — Every `grade.sh` sources shared `lib/grade.sh` (assertion helpers: `assert_resource_exists`, `assert_field_eq`, `assert_pod_ready`, `assert_pvc_bound`, `assert_can_i`, `assert_egress_allowed`, `assert_endpoints_nonempty`) and `lib/traps.sh` (trap detection helpers) — no grader reimplements these inline.
- [ ] **GRADE-02** — Grader assertions are behavioural or structural, never "object exists + `grep`". Behavioural assertions use `kubectl auth can-i`, `kubectl exec ... -- <probe>`, `kubectl wait`, jsonpath filters with `?(@.name==...)`. CI lint fails any grader containing `kubectl get | grep` or `kubectl get -A`.
- [ ] **GRADE-03** — Every question emits a stable `SCORE: <n>/<max>` line and at least one `Trap N: <name>: <description>` line per detected failure mode. Exit 0 = pass; exit 1 = fail with at least one trap emitted.
- [ ] **GRADE-04** — Every question ships ≥3 named traps in front-matter; each trap references a canonical ID registered in `traps/catalog.yaml`.
- [ ] **GRADE-05** — The trap catalog (`traps/catalog.yaml`) seed-includes the 8 CONCERNS.md-derived content-bug traps: `pss-error-string-mismatch`, `psp-fictional-pod-label-exemption`, `kubelet-runtime-flag-in-kubeconfig`, `removed-container-runtime-flag`, `hostpath-pv-without-nodeaffinity`, `as-flag-format-wrong`, `default-sa-used`, `missing-dns-egress`.
- [ ] **GRADE-06** — Round-trip self-check: `setup.sh && grade.sh` (no candidate solution applied) must FAIL with ≥1 trap emitted; `setup.sh && bash <reference-solution> && grade.sh` must PASS. Verified in CI for every question.

### Domain packs — 5 packs covering the v1.35 syllabus

- [ ] **PACK-01** — Storage pack (10% weight): at least one question per uncovered Study Progress Tracker checkbox in this domain, including a question for CSI / `VolumeSnapshot` (CG-01) and `WaitForFirstConsumer` semantics. Each pack ships a `manifest.yaml` listing its questions and a `README.md` with the disclaimer.
- [ ] **PACK-02** — Workloads & Scheduling pack (15%): includes native-sidecar question (CG-08) and metrics-server bootstrap question (CG-06) as HPA prereq.
- [ ] **PACK-03** — Services & Networking pack (20%): includes kube-proxy mode inspection (CG-15) and NetworkPolicy `endPort` (CG-16); references existing Gateway API / Ingress exercises as link-only prior art.
- [ ] **PACK-04** — Cluster Architecture pack (25%): includes PSS replacement (CG-10) with correct v1.25+ error wording, audit-policy question (CG-11), CRI-dockerd replacement with correct `--container-runtime-endpoint` flag and correct `/var/lib/kubelet/kubeadm-flags.env` file (CG-13), CRD basics (CG-12).
- [ ] **PACK-05** — Troubleshooting pack (30%, largest): at least one question each for CoreDNS (CG-03), `kubectl debug node` (CG-04), NetworkPolicy troubleshooting (CG-05); references existing troubleshooting playbook as link-only.
- [ ] **PACK-06** — Every question declares front-matter: `id`, `domain`, `estimatedMinutes ∈ [4, 12]`, `verified_against: "1.35"`, `traps: []` (≥3 IDs), `references: []` (link-only to existing exercises / skeletons).
- [ ] **PACK-07** — Every pack's questions collectively map 1-to-1 against the v1.35 Study Progress Tracker checkboxes for that domain (coverage-matrix lint enforces).

### Mock-exam packs — two timed blueprints

- [ ] **MOCK-01** — `exams/blueprint-alpha/manifest.yaml` composes exactly 17 questions by `pack/slug` reference (no duplication), weighted 10 / 30 / 15 / 25 / 20 to match the v1.35 CKA blueprint. Sum of per-question `estimatedMinutes` is between 110 and 120.
- [ ] **MOCK-02** — `exams/blueprint-bravo/manifest.yaml` composes a different 17-question draw from the same packs, same weighting, same time budget — so a candidate can retake without facing identical questions.
- [ ] **MOCK-03** — Both blueprints include the mandatory independent-content disclaimer ("Not real CKA exam content; independently authored") in their manifest `README.md`.

### Score report

- [ ] **REPORT-01** — At exam end, `cka-sim` writes a Markdown report to `~/.cka-sim/sessions/<ts>.md` containing: total score / 100, pass / fail vs the 66% mark, per-domain percentage table sorted lowest first, trap-frequency table (top 5 traps with counts), and a "Suggested next drills" section derived from failed-question domain tags.
- [ ] **REPORT-02** — `cka-sim score [<ts>]` re-displays any past session's report; `cka-sim list history` shows all completed sessions.

### Supersede existing study-guide content (policy, no deletion)

- [ ] **BANNER-01** — Add a 6-line banner block at the top of `exercises/README.md`, `mock-exams/README.md`, and the root `README.md` pointing at `cka-sim/`. Existing content is not modified below the banner.
- [ ] **BANNER-02** — Existing exercises remain valid link-only references from new pack `metadata.yaml.references[]`; no `setup.sh` ever sources or executes anything under `exercises/`.

### Documentation

- [ ] **DOC-01** — `cka-sim/README.md` documents quickstart: `bootstrap` → `doctor` → `drill` → `exam`. Lives at `cka-sim/README.md` so it doesn't conflict with the root README.
- [ ] **DOC-02** — `cka-sim/AUTHORING.md` documents how to add a new question, including the triplet template, the trap-registration flow, and the `metadata.yaml` front-matter schema.
- [ ] **DOC-03** — `cka-sim/SCHEMA.md` gives the canonical YAML schema for `metadata.yaml`, pack `manifest.yaml`, exam blueprint `manifest.yaml`, and `traps/catalog.yaml`.
- [ ] **DOC-04** — Root-level `CONTRIBUTING.md` gains a new "Authoring exam-sim questions" section pointing to `cka-sim/AUTHORING.md` and restating the CNCF-no-real-exam-content policy.

### CI integration

- [ ] **CI-01** — `scripts/validate-local.sh` extends to walk `cka-sim/**/*.yaml` with yamllint and `cka-sim/**/*.sh` with shellcheck.
- [ ] **CI-02** — `.github/workflows/validate.yml` extends `paths:` to include `cka-sim/**`, and adds a shellcheck step. Adds a "deprecated strings" grep lint (fails PR on `PodSecurityPolicy`, `--container-runtime=remote`, `policy/v1beta1`, `gitRepo:`, `dockershim`).
- [ ] **CI-03** — Pack lint: fails on `kubectl get | grep`, `kubectl get -A`, missing `verified_against: 1.35`, missing ≥3 traps, unregistered trap IDs, cluster-scoped-name collisions across questions in a pack.

## Out of Scope (v1 — explicit exclusions)

Each mirrored from PROJECT.md "Out of Scope" plus FEATURES.md "Anti-Features". Do not re-add without a milestone-level discussion.

- VM provisioning (Terraform / `gcloud compute instances create`)
- `kubeadm init/join` bootstrap
- Browser / PSI-Bridge web-terminal emulation
- CKAD / CKS / KCNA content or v1.34-and-earlier backports
- Multi-tenant / shared-cluster mode, auth, leaderboards
- Cloud-vendor-specific labs (GKE/EKS/AKS features)
- killer.sh-style portal / VPN / remote infra
- Real CKA exam content (CNCF NDA)
- GUI / web portal
- Auto-grader for fenced YAML inside the old `exercises/` READMEs (old content is reference-only)
- CONCERNS.md content fixes inside `exercises/` / `mock-exams/` (existing content stays as superseded; content bugs are encoded as traps, not retroactively fixed)
- Retrofitting or rewriting the 31 existing exercises
